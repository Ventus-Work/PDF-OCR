## 1. 목적

Phase15-Step6까지의 FastAPI + React/Vite MVP는 PDF 업로드, 실행, 결과 다운로드, QA 확인을 제공한다.

Step7의 목적은 실제 반복 운영 화면으로 쓰기 위해 다음 기능을 추가하는 것이다.

- 단일 PDF뿐 아니라 PDF 폴더 단위 batch 실행
- 완료된 작업의 로컬 output 폴더 바로 열기
- JSON, Markdown, Excel 산출물을 다운로드 전에 웹에서 읽기 전용으로 리뷰
- AI API 호출, 토큰 사용량, 일별/월별 누계를 확인하는 사용량 대시보드
- 현재 카드형 MVP를 운영형 대시보드 UI로 정리

## 2. 구현 범위

주요 구현 범위는 다음과 같다.

- `POST /api/jobs/batch`: 여러 PDF를 하나의 job으로 실행
- `POST /api/jobs/{job_id}/open-folder`: 로컬 job 폴더 열기
- `GET /api/jobs/{job_id}/artifacts/{artifact_id}/preview`: 산출물 preview
- `GET /api/usage/summary`, `GET /api/usage/daily`, `GET /api/usage/jobs/{job_id}`: 사용량 조회
- 프론트 업로드 패널, 결과 탭, 리뷰 탭, 사용량 탭, 작업 상태 패널 개선
- `output/ui_usage/usage.db` 기반 사용량 저장소

## 3. 설계 원칙

- 기존 Step6의 단일 PDF job 흐름은 유지한다.
- batch job은 기존 CLI의 디렉토리 입력 모드를 활용한다.
- 작업폴더 열기는 로컬 Windows 업무 도구 기능으로 보고, `output/ui_runs/<job_id>/` 내부만 허용한다.
- 산출물 리뷰는 편집 기능이 아니라 읽기 전용 preview로 제한한다.
- `.env`와 API key 원문은 읽거나 저장하거나 화면에 표시하지 않는다.
- 사용량 저장은 파서 실행을 실패시키지 않는 보조 기록으로 처리한다.

## 4. API 상세

### 4.1 Batch Job

Endpoint:

```text
POST /api/jobs/batch
```

입력:

- `files`: PDF 파일 목록
- `preset`
- `engine`
- `pages`
- `bom_fallback`
- `no_cache`

처리:

- 각 PDF를 `output/ui_runs/<job_id>/input/`에 저장한다.
- 파일명이 중복되면 `_2`, `_3` suffix로 충돌을 피한다.
- `input_path`는 단일 파일이 아니라 input 폴더를 가리킨다.
- 기존 CLI는 폴더 입력 시 batch 모드로 동작한다.

완료 기준:

- 여러 PDF 선택 시 하나의 job으로 생성된다.
- `JOB_STATUS.json`에 `input_count`가 기록된다.
- 최근 작업 목록에서 입력 파일 수가 보인다.

### 4.2 작업폴더 열기

Endpoint:

```text
POST /api/jobs/{job_id}/open-folder
```

처리:

- job 폴더가 `output/ui_runs/` 안에 있는지 확인한다.
- Windows 로컬 환경에서는 Explorer로 폴더를 연다.
- 테스트나 비지원 환경에서는 폴더 경로만 반환한다.

완료 기준:

- 완료, 실패, 취소된 job에서 `작업폴더 열기` 버튼이 활성화된다.
- path traversal 시도가 차단된다.

### 4.3 산출물 Preview

Endpoint:

```text
GET /api/jobs/{job_id}/artifacts/{artifact_id}/preview
```

지원 확장자:

- `.json`
- `.md`
- `.xlsx`

처리:

- JSON은 parse 후 pretty text와 구조 데이터를 반환한다.
- Markdown은 UTF-8 원문을 반환하고 프론트에서 안전하게 렌더링한다.
- Excel은 `openpyxl`로 첫 시트의 header와 상위 100행을 반환한다.

완료 기준:

- 결과 탭에서 `보기`와 `받기`가 분리된다.
- 리뷰 탭에서 JSON, MD, Excel을 다운로드 없이 확인할 수 있다.

### 4.4 사용량 대시보드

저장소:

```text
output/ui_usage/usage.db
```

Endpoint:

```text
GET /api/usage/summary?range=today|month|all
GET /api/usage/daily?month=YYYY-MM
GET /api/usage/jobs/{job_id}
```

기록 필드:

- timestamp
- job_id
- engine
- provider
- model
- input_tokens
- output_tokens
- total_tokens
- estimated_cost_usd
- status
- token_status

정책:

- Gemini는 usage metadata의 토큰을 기록한다.
- Z.ai/Mistral은 토큰이 확인되지 않으면 호출 수와 `unknown` token status를 기록한다.
- usage 기록 실패는 파서 실행 실패로 전파하지 않는다.

## 5. UI 상세

### 5.1 업로드 패널

- `PDF 파일 선택` 버튼을 제공한다.
- `PDF 폴더 선택` 버튼을 제공한다.
- 드래그 드롭은 단일/복수 PDF를 모두 받는다.
- 선택된 파일이 1개면 파일명을 표시한다.
- 여러 개면 PDF 개수와 총 용량을 표시한다.

### 5.2 작업 상태 패널

- 기존 상태, 엔진, 종료 코드 표시를 유지한다.
- 완료/실패/취소 job에는 `작업폴더 열기` 버튼을 추가한다.

### 5.3 결과/리뷰

- 결과 탭은 산출물 목록과 `보기`, `받기` 버튼을 제공한다.
- 리뷰 탭은 선택된 산출물 preview를 표시한다.
- Excel은 표 preview, JSON은 pretty text, Markdown은 간단한 preview 렌더링으로 표시한다.

### 5.4 사용량

- `사용량` 탭을 추가한다.
- 전체 누적 호출 수, 입력 토큰, 출력 토큰, 예상 비용을 표시한다.
- 엔진별 누적 표를 표시한다.
- 일별 토큰 사용량과 현재 선택 job의 사용량 이벤트를 표시한다.
