# Phase15-Step5 통합 검증 운영 상세 구현 기술서

작성일: 2026-04-27

대상 프로젝트: `Project/PJT_A/ps-docparser`

## 1. 목적

FastAPI API와 React UI가 기존 `ps-docparser` 백엔드 기준선을 깨지 않고 동작하는지 검증한다.

Step5는 구현 완료 후 실행할 검증, 운영 명령, 장애 판단 기준을 고정한다.

## 2. 검증 대상

- 기존 백엔드 회귀
- FastAPI endpoint
- job subprocess 실행
- artifact/QA 조회
- React build
- 업로드-실행-다운로드 E2E
- 실측 output QA 표시

## 3. 제외 범위

- 원격 배포 smoke
- 부하 테스트
- 브라우저 자동화 전체 시나리오
- 멀티 유저 race condition
- 장시간 대형 PDF full run

## 4. 로컬 실행 명령

API 서버:

```powershell
cd Project/PJT_A/ps-docparser
uvicorn api.app:app --host 127.0.0.1 --port 8000 --reload
```

프론트 개발 서버:

```powershell
cd Project/PJT_A/ps-docparser/frontend
npm install
npm run dev
```

프론트 빌드:

```powershell
cd Project/PJT_A/ps-docparser/frontend
npm run build
```

Google Drive 동기화 경로에서는 `frontend/node_modules` 직접 설치가 파일 잠금으로 실패할 수 있다. 이 프로젝트의 기본 운영 명령은 아래 우회 스크립트다.

```powershell
cd Project/PJT_A/ps-docparser
.\scripts\frontend_local.ps1 dev
.\scripts\frontend_local.ps1 build
```

이 스크립트는 `frontend/` 소스를 `%LOCALAPPDATA%\Codex\ps-docparser-frontend-work`로 복사하고, npm 설치/빌드/개발 서버 실행을 Drive 밖에서 수행한다.

## 7. E2E 수동 검증

샘플:

```text
00_견적서_원본/53-83 OKOK.pdf
```

권장 옵션:

```text
preset=pumsem
engine=auto
pages=1-10
output=excel
```

검증 순서:

1. UI에서 PDF 업로드
2. preset/engine/pages 선택
3. 실행 시작
4. running 상태 확인
5. succeeded 상태 확인
6. 결과 탭에서 md/json/xlsx 확인
7. QA 탭에서 status 확인
8. 다운로드 버튼으로 파일 다운로드
9. Manifest 탭에서 representative/diagnostic 구조 확인

## 8. 실측 QA 표시 검증

기준 리포트:

```text
output/실측테스트_20260427_baseline_lock/OUTPUT_QA_REPORT.md
```

UI 표시 필수 항목:

```text
Status: WARN
JSON files: 12
Excel files: 12
RUN_MANIFEST.json: yes
Manifest inputs: 11
Manifest representative: 11
Manifest diagnostic: 1
Header/key mismatch: 0
Bad composite headers: 0
Quality warnings: empty_tail_column 1
Manifest domains: bom 8, estimate 2, pumsem 1
```

판정:

- `WARN`은 실패로 표시하지 않는다.
- `Header/key mismatch > 0`이면 강한 경고다.
- `Bad composite headers > 0`이면 강한 경고다.
- `empty_tail_column 1`은 현재 기준선의 알려진 warning이다.

## 9. 운영 파일 관리

생성 위치:

```text
output/ui_runs/
frontend/dist/
```

Git 정책:

- `output/`은 Git 추적 대상 아님
- `frontend/dist/`는 v1에서 Git 추적하지 않음
- `.env`, `.coverage`, 로그 파일은 커밋 금지

커밋 전 확인:

```powershell
git status --short
git diff --check
git check-ignore -v ps-docparser/output/ui_runs
```

## 10. 실패 대응 기준

- API tests 실패: UI 구현 전에 API 수정
- React build 실패: frontend만 수정

- 기존 `pytest` 실패: Phase15 변경을 의심하고 백엔드 회귀부터 확인
- E2E job 실패: log tail과 CLI 직접 실행 결과 비교
- QA report 없음: analyzer 실행 여부 확인
- 다운로드 실패: artifact path mapping과 traversal 검사 확인
