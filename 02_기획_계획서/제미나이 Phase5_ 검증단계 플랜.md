# Phase 5 상세 작업 계획서 (액션 플랜)

## 📌 결론 (목표 및 방향성)
Phase 5의 핵심 목적은 기존 구현된 파싱/BOM 추출 엔진을 **API 비용 절감 방안을 통해 프로덕션화**하고, **일괄 배치 처리와 수량 자동 집계**를 도입하여 실무에 바로 적용 가능한 최종 산출물 파이프라인을 완성하는 것입니다. 

사용자님의 '기능 단위 분할(Agile)' 및 '단위별 완벽 구현(Waterfall)' 원칙에 따라 Phase 5 과제를 5개의 독립된 실행 단위로 쪼개어 제안합니다.

---

## 🛠️ 세부 단위 분할 계획 (Agile Plan)

> **진행 원칙**: 각 단위를 순차적으로 진행하며, 앞 단위가 완벽히 동작하고 검증된 후에 다음 단계로 넘어갑니다.

### 단위 1: API 캐싱 레이어 구축 (비용 최적화)
- **목적**: 동일 대상(이미지/페이지)에 대한 반복 API 호출에 따른 비용 및 시간 낭비 방지
- **상세 작업**:
  - `cache/table_cache.py` 구현 (SQLite 기반 로컬 캐싱).
  - 식별값(Key) 생성 로직 마련: `sha256(이미지 바이너리 + 엔진 설정)`.
  - TTL (ex: 30일) 관리 및 캐시 조회/저장 핸들러 구성.
  - `extractors/` 및 `engines/` 에 캐시 레이어 주입 및 유닛 테스트(캐시 적중률 확인).

### 단위 2: 배치 처리(Batch) 통합 및 기능 강화
- **목적**: 별도의 `batch_test.py` 스크립트가 아닌, 표준 파이프라인(`main.py`)에서 폴더 단위 일괄 실행 지원.
- **상세 작업**:
  - `main.py`에 CLI 인수 (`--batch`, `--input-dir` 등) 통합.
  - 폴더 내 모든 대상 PDF 순회 및 진행 상태(Progress Bar) CLI 출력.
  - 최종 요약 리포트(성공/실패 내역, 소요 시간, 토큰 사용량/캐시 적중 수) 생성 기능.

### 단위 3: BOM 집계 자동화 모듈 구현 (핵심 기능)
- **목적**: 단순히 읽은 테이블을 저장하는 것을 넘어, 같은 속성을 가진 자재들의 수량을 자동 합산하여 최종 견적 베이스로 가공.
- **상세 작업**:
  - 그룹핑 조건 정의: `MATERIAL`, `SIZE`, `THK`/`SCH` 등 동일 스펙.
  - 단일 대상들의 `Q'TY`를 더하고, 이에 따른 `TOTAL WEIGHT` 재산정 로직 (가칭 `exporters/bom_aggregator.py` 또는 파서 확장).
  - 집계 처리 전후의 수치 무결성 검증 유닛테스트 구성.

### 단위 4: Excel Exporter 템플릿 연동 완성
- **목적**: 추출된 데이터가 견적서 양식이나 포맷이 정리된 지정된 Excel 템플릿에 출력되도록 구현.
- **상세 작업**:
  - 프로젝트 내 `templates/견적서_양식.xlsx` 양식 정의 및 템플릿 디렉토리 준비.
  - `excel_exporter._write_preset_sheets()` 의 미완성 분기 로직(배관 등 프리셋 조건) 구현.
  - *부가 작업 (선택)*: 시간적 여유 시 PR2/FP(배관 도면, Filter Press)에 대한 키워드 추출 프리셋 뼈대 추가.

### 단위 5: 61개 파일 대상 최종 검증 테스트
- **목적**: 프로덕션 배포 전, 수집된 61종 전체 도면(BOM) PDF에 대한 성공률 담보.
- **상세 작업**:
  - 단위 1~4가 모두 결합된 `main.py --batch` 파이프라인 가동.
  - 도면 60여 개의 TOTAL WEIGHT/Q'TY가 실제와 일치하는지 샘플링 및 교차 검증.
  - 파급 효과 및 오류 패턴 분석, 최종 디버깅.

## ❓ 논의 필요 사항

1. **GUI 개발 우선순위 검토**: 요약 보고서에는 `gui.py` 구축 항목이 하위 우선순위로 있습니다. 당장 CLI 형태로 업무 투입이 가능하다면 GUI는 후순위로 미뤄둘까요, 아니면 단위 5 이후 곧바로 착수할까요?
2. **템플릿 양식 제공 여부**: 단위 4에서 참조할 `견적서_양식.xlsx`는 기존에 사용하시던 특정 엑셀 폼(수식 등 포함)을 적용할 계획이신가요? (그렇다면 해당 양식의 작업/제공이 우선되어야 합니다.)
