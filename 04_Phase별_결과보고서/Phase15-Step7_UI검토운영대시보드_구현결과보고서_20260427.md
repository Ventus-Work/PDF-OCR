- JSON, Markdown, Excel 산출물 웹 preview

- 산출물 `보기`를 화면 하단 탭 이동이 아니라 팝업 모달로 표시

## 4. 구현 상세

### 4.1 폴더 선택 Batch

프론트에서 `PDF 폴더 선택`으로 여러 PDF를 선택하면 `POST /api/jobs/batch`로 전송한다.

백엔드는 모든 PDF를 `output/ui_runs/<job_id>/input/`에 저장하고, CLI 입력 경로를 input 폴더로 넘긴다. 기존 `main.py`의 디렉토리 입력 batch 모드를 그대로 사용한다.

### 4.2 작업폴더 열기

작업 상태 패널에 `작업폴더 열기` 버튼을 추가했다.

백엔드는 job 폴더가 `output/ui_runs/` 내부인지 확인한 뒤 Windows 로컬 환경에서는 Explorer를 연다. 테스트 환경이나 비지원 환경에서는 폴더 경로만 반환한다.

### 4.3 산출물 리뷰

결과 탭에서 `보기`를 누르면 현재 화면 위에 산출물 리뷰 팝업이 열린다.

- JSON: pretty text와 구조 데이터 반환
- Markdown: 제목, 페이지 마커, HTML table을 읽기 전용 viewer로 표시하고 원문 보기를 별도 제공
- Excel: `openpyxl`로 첫 시트 상위 100행 preview

다운로드 기능은 기존처럼 유지한다.

팝업은 닫기 버튼, 바깥 영역 클릭, Esc 키로 닫을 수 있다.

### 4.4 사용량 대시보드

`UsageTracker`는 UI job 하위 프로세스에서 `PS_DOCPARSER_JOB_ID`, `PS_DOCPARSER_USAGE_DB` 환경변수를 받으면 SQLite에 사용량 이벤트를 기록한다.

기록 정책:

- Gemini는 실제 input/output token을 기록한다.
- Z.ai/Mistral은 토큰이 확인되지 않으면 `token_status=unknown`으로 호출 수를 기록한다.
- usage 저장 실패는 파서 실행 실패로 전파하지 않는다.

프론트 사용량 탭은 다음을 표시한다.

- 전체 API 호출 수
- 입력/출력 토큰
- 예상 비용
- 엔진별 누적
- 일별 사용량
- 현재 선택 job의 사용량 이벤트

### 5.6 견적서 Excel 정합성 및 가독성 복구

적용일: 2026-04-27

대상 run:

```text
output/ui_runs/ui_20260427_141100_d12dec32
```

확인된 문제:

- 자재 견적서가 `generic`으로 라우팅되어 Excel이 `Table`, `Table_2` 중심으로 생성됐다.
- 품목표 아래의 `-이하 여백-`, 납기, 운송조건, 결제조건, 착지, 인도조건 행이 품목표 데이터 안에 섞였다.
- 반복되는 긴 조건 문구가 여러 셀에 중복 출력되어 Excel 첫 화면에서 내용이 잘려 보였다.
- 업체 정보 표가 원본 HTML colspan 구조 영향으로 의미 있는 세로형 표가 아니라 중복 가로형 표처럼 보였다.
- 기존 QA는 header/key mismatch는 잡았지만, `generic` 오분류와 반복 장문 셀은 경고하지 못했다.

적용 내용:

- `detector.py`의 자재 견적서 판별을 보강해 `품목`, `수량`, `단가`, `공급가액` 계열 표가 있으면 `estimate` 후보로 라우팅한다.
- `DocumentPipeline`에서 `material_quote`를 generic 강제 조건으로 보지 않도록 수정했다.
- 견적서 출력 계약 정규화에서 업체 정보 표를 `업체정보` 시트로 분리한다.
- 품목표의 `-이하 여백-` 이후 조건 행을 `조건` 시트로 분리하고, 반복 장문 비고는 첫 행에만 남긴다.
- 모든 값이 비어 있는 tail column은 Excel/JSON export 전에 제거한다.
- QA에 `generic_estimate_misroute_suspected`, `repeated_long_cell_value` 경고를 추가해 같은 문제가 다시 숨지 않게 했다.
- generic/조건 계열 Excel 시트에 freeze pane, auto filter, 동적 row height를 적용해 첫 화면 가독성을 개선했다.

## 6. 남은 위험

- 실제 PDF 단일 업로드와 batch smoke는 수행했다. 다만 batch는 비용을 줄이기 위해 `local` 엔진으로 검증했으므로, 최종 잠금 전에는 필요한 대표 샘플만 `gemini` 또는 `zai`로 추가 검증할 수 있다.
- `작업폴더 열기`는 로컬 Windows에서 `opened=true`로 확인했다. 원격 배포 시에는 비활성화하거나 경로 복사 기능으로 대체해야 한다.
- Z.ai/Mistral의 토큰 수는 현재 SDK 응답에서 안정적으로 확인하지 못해 호출 수 중심으로 기록한다.
- 단일 PDF smoke와 batch smoke 모두 `repeated_long_cell_value` warning이 남았다. 값 손실은 아니지만 화면 가독성/중복 텍스트 후처리 대상으로 계속 추적한다.
- 현재 워크트리에는 백엔드 안정화, Phase15 Step6, Step7, 문서 변경이 섞여 있다. 커밋 전 변경 단위 분리가 필요하다.
- `.env`는 열람하지 않았다. 파일명 기준 점검에서는 `.env`가 ignore 대상이고, Git 추적 secret 후보는 `.env.example`만 확인했다.

## 7. 커밋 단위 분리안

실제 커밋은 수행하지 않았다. 커밋/푸시는 plan-required operation이므로, 아래 단위로 분리한 뒤 사용자 확인 후 진행한다.

권장 커밋 단위:
