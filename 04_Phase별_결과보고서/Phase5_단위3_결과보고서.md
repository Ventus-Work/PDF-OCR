# Phase 5: 단위 3 (BOM 서포트 집계기) 구현 결과 보고서

## 1. 개요
본 보고서는 `ps-docparser`의 **Phase 5 단위 3 (BOM 집계기 모듈 구현 및 검증)** 결과를 요약합니다.
이 과정을 통해 61개 이상의 도면에 산발적으로 나뉘어 있는 배관 서포트 재료 내역을 `(SIZE, MATERIAL)` 기준으로 병합하여 유의미한 전체 수량 및 중량 정보로 요약할 수 있게 되었습니다.

---

## 2. 주요 구현 내역 (코드베이스 수정)

| 생성/수정 파일 | 상세 내용 |
| --- | --- |
| **[NEW] `exporters/bom_aggregator.py`** | **배치용 BOM 집계 모듈**<br>- **헤더 정규화 (`_HEADER_ALIASES`)**: 개별 도면별로 다르게 OCR된 컬럼명 (예: `WT(KG)`, `WT(kg)`, `중량` 등)을 대표 키스톤으로 정규화하여 탐색하는 기능을 구현했습니다.<br>- **안전한 형변환 (`_parse_float`)**: `'1,000'`, `'25EA'`, `'10kg'` 처럼 단위나 콤마가 혼재된 문자열에서 실수(float)만 안전하게 추출하여 변환 실패(Crash) 및 결측치 오류를 원천 차단했습니다.<br>- **그룹화 및 합산 (`aggregate_boms`)**: 모든 파싱된 JSON 파일을 순회하여 SIZE와 MATERIAL이 동일한 부품끼리 묶고, 최종적으로 단일 "BOM" 테이블 섹션(`Array` 형식)으로 재압축(Export)하도록 설계되었습니다. |
| **[NEW] `test_phase5_unit3.py`** | **단위 3 독립 검증 도구**<br>- 의존성 없는 환경에서 `_parse_float`, 헤더 매핑 규칙, 다중 도면 그룹핑 로직을 검증하는 22개 테스트케이스를 작성하여 안정성을 증명했습니다. |
