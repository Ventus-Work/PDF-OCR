# Phase 14: BOM 도면 메타데이터 추출기 결과 보고서

## 1. 개요
Phase 14 "BOM 도면 메타데이터 추출기 구현"이 완료되었습니다. 
도면 내 BOM 영역 상단에 기재되는 타이틀 블록(DWG NO, REV, TITLE 등)을 OCR 결과에서 추출하여 엑셀 시트로 분리 저장하는 파이프라인 연동이 적용되었습니다.

## 2. 세부 구현 사항

### 2.1 추출 모듈 신규 구현 (Step 14-1)
- `extractors/drawing_meta.py`: 도면 타이틀 블록을 파싱하는 전용 추출기 작성.
- 키워드(DWG NO, REV, TITLE 등 11개 필드) + 구분자 정규식을 통한 **first-match-wins** 전략.
- 파이프(`|`) 인라인 테이블 분리, HTML 태그 정규화, 연속 공백 압축 및 빈 값(None) 필터링 로직 구현.
- `extractors/bom_types.py`: `BomExtractionResult` 모델에 `drawing_metadata` 필드(dict) 추가.

### 2.2 OCR 재시도 모듈 연동 (Step 14-2)
- `extractors/bom_ocr_retry.py`: 1차 Full-page OCR 처리 직후 확정된 `raw_text`를 `extract_drawing_meta` 함수로 전달.
- 도면 메타데이터가 파이프라인의 데이터 흐름에 자동으로 누적되도록 반영.

### 2.3 JSON 변환기 연동 (Step 14-3)
- `extractors/bom_converter.py`: `BomExtractionResult.drawing_metadata`에 1개 이상의 유효한 값이 존재할 경우, JSON 섹션 리스트의 맨 앞에 `DRAWING-META-1` 섹션을 추가(Prepend).

### 2.4 Excel 내보내기 연동 (Step 14-4)
- `exporters/excel_builders.py`: `_build_drawing_meta_sheet` 빌더 추가.
  - A열 필드명, B열 데이터 형태의 수직 테이블로 시트 렌더링.
- `exporters/excel_exporter.py`: `drawing_meta` 타입 섹션이 발견될 경우 워크북에 **"도면_메타"** 시트를 생성하고 빌더 호출.
