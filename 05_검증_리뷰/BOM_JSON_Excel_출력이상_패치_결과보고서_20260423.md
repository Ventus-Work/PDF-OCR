# BOM JSON / Excel 출력 이상 패치 완료 보고서

요청하신 BOM 파이프라인의 복합 헤더(2행 헤더) 누락 및 시트명 이슈에 대해 P1~P4 패치를 성공적으로 완료하였습니다.

## 변경 사항 요약

### 1. P1: 중복 헤더 dedupe 방어 (`bom_converter.py`)
- **이전**: 중복된 헤더명을 가진 셀이 있으면, JSON 변환 시 Dict의 Key 특성 상 뒤쪽 셀이 앞쪽 셀을 덮어씌움 (예: `WEIGHT` 2개가 있으면 마지막 값만 남음).
- **이후**: `bom.headers`를 순회하며 중복 여부를 감지하여 `WEIGHT`, `WEIGHT_2`, `WEIGHT_3` 과 같이 번호 Suffix를 추가하도록 변경.

### 2. P2: 다단 헤더(복합 헤더) 병합 (`bom_table_parser.py`)
- **이전**: `effective_grid[0]`만을 헤더로 보고 바로 다음 행부터는 무조건 데이터 행으로 편입시켜, 2번째 헤더 행("UNIT", "WEIGHT" 등)이 데이터처럼 출력되는 현상.
- **이후**: `effective_grid[1]`에 보조 헤더 키워드("UNIT", "WEIGHT", "LOSS", "M2", "KG", "단위" 등)가 2개 이상이고 숫자 비율이 절반 이하인 경우 해당 행을 **2차 헤더**로 간주합니다. 이를 바탕으로 `상위헤더 | 하위헤더` 형태로 문자열을 병합(`자재중량 [Kg] | WEIGHT`)하여 단일 헤더화 합니다.

### 3. P3: generic BOM 시트명 의미화 (`excel_classifier.py`, `excel_exporter.py`)
- **이전**: 테이블 타입이 일반 견적서 양식이 아닐 때 무조건 `"generic"` 타입으로 분류되어 "Table" 이라는 모호한 시트명이 부여됨.
- **이후**: 테이블 타입이 BOM이거나 헤더에 BOM 관련 주요 키워드가 감지되면 `"bom_generic"`으로 세분화하여, 엑셀 시트 생성 시 `"Table"` 대신 **`"BOM_자재표"`** 라는 직관적인 이름으로 부여합니다. 복수 개의 경우 `"BOM_자재표_2"` 로 자동 넘버링됩니다.

### 4. P4: BOM 프리셋 경고 로그 추가 (`bom_pipeline.py`)
- **이전**: 혼합 문서(견적/내역서 + BOM 표) 처리 시, 일부 테이블만 추출되어도 사용자가 그 원인을 즉각적으로 인지하기 어려움.
- **이후**: 출력 파이프라인 진행 시, 메타데이터 추출은 실패했으나 전체 원문 텍스트 길이가 1000자 이상이면서도 추출된 테이블은 극소수일 경우 **"⚠ 혼합 문서(견적/내역서+BOM 표 1~2개)로 추정됩니다. 도면 메타데이터가 없고 텍스트가 많아 일부 테이블만 추출되었을 수 있습니다."** 라는 안내(Warning 로그)가 표출됩니다.
