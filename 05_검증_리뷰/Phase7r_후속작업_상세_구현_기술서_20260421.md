## C. CI `slow` marker — Mistral 테스트 격리

### 현황 및 목표

`test_mistral_engine.py` 의 5개 케이스가 각각 2~5초 소요(MistralAI SDK 임포트 지연). CI 에서 전체 `pytest tests/unit/` 러닝 시 25초 이상 블로킹.

`pytest.ini` 에 `slow` 마커가 이미 정의되어 있으므로, **마커 추가 + addopts 수정**만 하면 된다.

### 3-1. 대상 파일 및 추가 내용

**파일**: `ps-docparser/tests/unit/engines/test_mistral_engine.py`

모든 클래스 또는 개별 케이스에 `@pytest.mark.slow` 추가.
클래스 레벨 데코레이터가 더 간결:

```python
import pytest

@pytest.mark.slow
class TestMistralEngineInit:
    ...

@pytest.mark.slow
class TestMistralEngineOcrDocument:
    ...

@pytest.mark.slow
class TestMistralEngineOcrImage:
    ...
```

### 3-2. `pytest.ini` 기본 옵션 수정

현재 `addopts` 에 `-m "not slow"` 가 없으므로 slow 테스트도 기본 러닝에 포함된다.

**파일**: `ps-docparser/pytest.ini`

```ini
addopts =
    --strict-markers
    --tb=short
    -v
    --durations=10
    -p no:cacheprovider
    -m "not slow"           # ← 추가
```

> **효과**: 기본 `pytest` 러닝에서 slow 테스트 자동 제외. 느린 테스트 포함 실행 시: `pytest -m slow` 또는 `pytest -m ""`.

### 3-3. 검증

```bash
# slow 제외 러닝 (기본)
python -m pytest tests/unit/ -q
# → 261 passed (Mistral 5개 제외), 5 deselected

# slow 포함 러닝
python -m pytest tests/unit/ -m "" -q
# → 266 passed
```

### 3-4. 커밋 메시지


## D. `google.generativeai` → `google.genai` 마이그레이션

### D-1. 배경 및 긴급도

```
FutureWarning: All support for the `google.generativeai` package has ended.
It will no longer be receiving updates or bug fixes.
Please switch to the `google.genai` package as soon as possible.
```

현재 설치 상태:
- `google.generativeai`: 설치됨, 경고만 발생 (기능 정상)
- `google.genai`: `1.63.0` 설치됨, 즉시 사용 가능

**긴급도**: 당장 동작 불가는 아니지만, 패치/보안 업데이트 없이 방치 시 향후 API 변경에 취약.

### D-2. API 변경 대조표

| 항목 | 구 API (`google.generativeai`) | 신 API (`google.genai`) |
|---|---|---|
| import | `import google.generativeai as genai` | `import google.genai as genai` |
| 초기화 | `genai.configure(api_key=key)` | `client = genai.Client(api_key=key)` |
| 모델 객체 | `model = genai.GenerativeModel(name)` | 없음 (client에 통합) |
| 생성 호출 | `model.generate_content([prompt, img], safety_settings=LIST)` | `client.models.generate_content(model=name, contents=[prompt, img], config=Config(...))` |
| safety_settings | `[{"category": "...", "threshold": "..."}]` (dict 리스트) | `types.GenerateContentConfig(safety_settings=[types.SafetySetting(...)])` |
| 응답 텍스트 | `response.text` | `response.text` (동일) |
| 토큰 (입력) | `response.usage_metadata.prompt_token_count` | `response.usage_metadata.prompt_token_count` (동일) |
| 토큰 (출력) | `response.usage_metadata.candidates_token_count` | `response.usage_metadata.candidates_token_count` (동일) |

> `response.text`, `usage_metadata` 필드명이 동일하므로 `parse_usage_metadata()` 함수는 **변경 없음**.

#### D-3-2. `__init__` 변경

```python
    try:
        import google.generativeai as genai

# After
def __init__(self, api_key, model="gemini-2.0-flash", tracker=None):
    try:
        import google.genai as genai
        from google.genai import types as genai_types
    except ImportError:
        raise ImportError(
            "Gemini 엔진을 사용하려면 google-genai 패키지가 필요합니다.\n"
            "설치: pip install google-genai"
        )

    if not api_key:
        raise ValueError(
            "Gemini 엔진에 GEMINI_API_KEY가 필요합니다.\n"
            ".env 파일에 GEMINI_API_KEY=your_key 를 추가하세요."
        )

    self._client = genai.Client(api_key=api_key)
    self._types = genai_types
    self.model_name = model
    self.tracker = tracker or UsageTracker()
    # safety_settings 사전 구성
    self._safety_settings = [
        genai_types.SafetySetting(category=cat, threshold=thr)
        for cat, thr in self.SAFETY_SETTING_DEFS
    ]
```

#### D-3-3. `extract_table` 변경

```python
# Before

    model = self._genai.GenerativeModel(self.model_name)
    time.sleep(FREE_TIER_DELAY)
    for attempt in range(max_retries):
        try:
            response = model.generate_content(
                [self.PROMPT_TABLE, image],
                safety_settings=self.SAFETY_SETTINGS,
            )
            ...

# After
def extract_table(self, image, table_num):
    time.sleep(FREE_TIER_DELAY)
    config = self._types.GenerateContentConfig(
        safety_settings=self._safety_settings
    )
    for attempt in range(max_retries):
        try:
            response = self._client.models.generate_content(
                model=self.model_name,
                contents=[self.PROMPT_TABLE, image],
                config=config,
            )
            ...
```

```python
"""tests/integration/test_hybrid_smoke.py — hybrid_extractor 실 PDF 스모크."""

from pathlib import Path
import pytest
from engines.local_engine import LocalEngine
from extractors.hybrid_extractor import process_pdf


@pytest.fixture(scope="module")
def minimal_pdf(sample_pdf_dir: Path) -> Path:
    p = sample_pdf_dir / "minimal.pdf"
    if not p.exists():
        pytest.skip("minimal.pdf 미생성 — scripts/create_minimal_pdf.py 실행 필요")
    return p


class TestHybridSmokeLocalEngine:
    def test_text_only_pdf_returns_nonempty_markdown(self, minimal_pdf):
        """
        텍스트 전용 PDF + LocalEngine → AI 미호출로 마크다운 반환.
        """
        engine = LocalEngine()
        md = process_pdf(str(minimal_pdf), engine=engine)
        assert isinstance(md, str)
        assert len(md) > 0

    def test_contains_expected_text(self, minimal_pdf):
        """
        PDF 에 삽입한 텍스트가 추출 결과에 포함되어야 한다.
        """
        engine = LocalEngine()
        md = process_pdf(str(minimal_pdf), engine=engine)
        assert "Test Document" in md or "Item" in md
```

## ZAI SDK Pydantic v1 호환성 관찰 방안 (부록)

현재 경고:

```
UserWarning: Core Pydantic V1 functionality isn't compatible with Python 3.14 or greater.
  from pydantic.v1.datetime_parse import (
```

**현 상태**: 경고만 발생, 동작 정상. ZAI SDK 내부 구현 문제이므로 직접 수정 불가.

**대응 방안**:
1. **버전 고정**: `requirements.txt` 에 `zai>=X.Y.Z` 형태로 현재 동작하는 버전 고정.
2. **경고 suppress** (임시): `pytest.ini` 에 `filterwarnings = ignore::UserWarning:zai` 추가.
3. **모니터링**: ZAI SDK 의 Pydantic v2 완전 마이그레이션 릴리스 주시, 업그레이드 시 테스트 재실행.
4. **실파손 시**: `pydantic.v1` 제거로 `ImportError` 발생하면 `zai` SDK 교체 또는 `engines/zai_engine.py` 를 직접 HTTP 호출로 교체.
