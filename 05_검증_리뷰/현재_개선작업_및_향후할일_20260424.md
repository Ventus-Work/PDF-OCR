### 3.1 BOM 복합 헤더 실측 보완

- 상태: `2026-04-24` 구현 반영
- `rowspan`으로 같은 헤더가 반복될 때 단일 헤더로 축약
- 1행/2행 헤더 길이 불일치 시 빈 헤더는 `Column_N`으로 명시
- 실측 BOM 문서를 회귀 fixture로 고정
- `DESCRIPTION | DESCRIPTION`, `DWG NO. | DWG NO.` 같은 중복 표현 제거
- `자재중량 [Kg] | UNIT` 같은 유의미한 복합 헤더는 유지
- 두 번째 헤더 행이 실제 데이터 행으로 유출되지 않도록 보장

### 3.2 데이터 행 정합성 보정

- 상태: `2026-04-24` 대표 fixture 기준 구현 반영
- `td` 부족, `colspan` 혼합, 행 길이 불균형으로 생기는 열 밀림 보정
- `expand_table`을 동적 grid 확장 방식으로 보완하여 rowspan 점유 슬롯 뒤의 추가 셀 유실 방지
- 앞쪽 빈 셀 누락 케이스를 우선 해결
- `수량`이 비고 `단위`에 숫자, 다음 UNIT 계열 열에 `식/EA/SET/LOT` 등이 들어간 실측 BOM 첫 row를 좁은 조건으로 좌측 보정
- `headers` 길이와 `rows[0].keys()` 정합성 맞춤
- 첫 행만 한 칸 밀리는 실측 BOM 케이스를 재현하고 수정 효과를 검증

- 상태: `2026-04-24` 일부 구현 반영

## 4. 향후 할 일

아래 항목은 즉시 핫픽스 범위가 아니라, 별도 Phase 또는 정책 정리가 필요한 작업들이다.

### 4.1 자동 폴백 / 듀얼 실행

- 현재 구현 상태:
  - `--bom-fallback auto|always|never`를 지원한다.
  - `--no-bom-fallback`은 호환용 alias로 유지한다.
  - `mixed_document` 또는 `material_quote` 신호에서 `estimate` 프리셋 자동 재실행을 수행한다.
  - PIPE처럼 primary BOM/LINE LIST가 강한 도면은 `auto`에서 fallback 생성을 줄인다.
  - 대표본은 기존 `_bom.*`를 유지하고, 보조본은 `_bom_fallback_estimate.*`로 추가 생성한다.
  - `RUN_MANIFEST.json`에서 대표본은 `role=representative`, fallback estimate는 `role=diagnostic`으로 기록한다.
- 남은 Phase:
  - 듀얼 실행 후 머지
  - estimate 결과의 대표본 승격 정책
  - 중복 섹션 제거와 결과 표현 방식 정리

### 4.2 `DWG NO.` table fallback 정책 검토

- 표 내부 `DWG NO.` 값을 메타 fallback으로 쓸지 여부 검토
- 기본 방향:
  - `non-empty unique 값이 정확히 1개`인 경우에만 제한적으로 허용
  - provenance 로그를 남겨야 함
- 현재 권장:
  - 정책 미확정
  - 기본 비활성 유지

### 4.3 구조 개선

- `excel_exporter.py`, `table_parser.py`, `bom_extractor.py` 대형 모듈 분해
- Poppler/Tesseract 경로 자동 감지 정리
- 정규식 캐싱, 벤치마크, CI 강화
- 장기적으로 500줄 초과 프로덕션 파일 0건을 목표로 분해

### 4.5 운영 고도화

### 4.6 도메인별 quality 계약 확장

- 현재 구현 상태:
  - BOM 외 estimate/pumsem/trade-statement/generic에 최소 `domain`, `role`, `quality`를 부여한다.
  - estimate/pumsem 일반 출력도 `RUN_MANIFEST.json` 대표본으로 기록한다.

  - `자재중량 [Kg] | UNIT` 같은 병합 헤더 유지
  - 두 번째 헤더 행이 데이터 행으로 유출되지 않음

### 5.2 데이터 행 정합성 보정

- 대상 위치:
  - `parsers/bom_table_parser.py`
  - `parsers/html_utils.py`의 `expand_table`
  - `extractors/bom_converter.py`
- 구현 원칙:
  - 데이터 행 길이를 `headers` 길이에 맞추되 단순 padding으로 끝내지 않음
  - `rowspan` 점유 슬롯 때문에 하위 행의 실제 열 수가 늘어나는 경우 grid를 동적으로 확장
  - 앞쪽 빈 셀 누락 케이스를 우선 해결
  - 변환 단계에서도 header보다 긴 row는 `Column_N` header를 추가해 데이터 유실 방지
- 기대 결과:
  - 첫 행 한 칸 밀림 현상 제거
  - `rows[0].keys()`와 `headers` 길이 일치
  - 실측 BOM 첫 행 정상 정렬

### 5.3 자동 폴백 / 듀얼 실행 기술 메모

- 대상 위치:
  - `pipelines/bom_pipeline.py`
  - 진입점 / 파이프라인 팩토리
- 현재 고정 정책:
  - CLI: `--bom-fallback auto|always|never`
  - 호환: `--no-bom-fallback`은 `never`와 동일
  - `mixed_document`: BOM/LINE LIST 합계 1~2개, 메타 없음, 본문 길이 1000자 이상
  - `material_quote`: 표 0개, 본문 길이 500자 이상, `detect_material_quote()` 참
  - 자동 폴백 시 대표본은 `_bom.*`, 보조본은 `_bom_fallback_estimate.*`
  - manifest 기준 대표본은 `role=representative`, 보조본은 `role=diagnostic`
  - batch aggregation은 manifest의 representative BOM JSON을 우선 사용한다
- 미구현 범위:
  - BOM과 estimate 동시 실행 후 머지
  - estimate 결과를 대표 산출물로 승격
  - 머지 우선순위와 중복 섹션 제거 기준

### 5.4 `DWG NO.` fallback 기술 메모

- 대상 위치:
  - `extractors/drawing_meta.py`
- 구현 조건:
  - 표 내부 `DWG NO.` 값의 non-empty unique 값이 정확히 1개일 때만 fallback 허용
  - `dwg_no source = table_fallback` 같은 provenance를 로그에 남길 것
- 기본 권장:
  - 정책 미확정
  - 기본 비활성 유지

### 5.5 품질 / 운영 기술 메모

- 테스트 목표:
  - 실측 fixture 기반 회귀셋 확대
  - 전체 커버리지 `75%+`
- CI 목표:
  - 핵심 파서, 파이프라인, Excel 렌더링 회귀 자동화
- 문서 목표:
  - `README.md`에 `output`, 대표 결과본, Drive 충돌 파일 대응 규칙 반영

## 6. 인터페이스 및 동작 영향

- `--bom-fallback auto|always|never` CLI 옵션이 추가되었다.
- `--no-bom-fallback`은 호환용 alias로 유지된다.
- 기본 동작에서는 `--preset bom` 결과 외에 `_bom_fallback_estimate.*` 보조 산출물이 추가될 수 있다.
- 실행 폴더에는 `RUN_MANIFEST.json`, `RUN_SUMMARY.md`가 생성될 수 있다.
- BOM JSON table에는 `domain`, `role`, `quality.status`, `quality.warnings`가 추가된다.
- 따라서 자동 폴백은 기능 추가가 아니라 정책 변경으로 취급해야 한다.
- `drawing_metadata` fallback은 정책 승인 전까지 기본 비활성으로 유지하는 것이 안전하다.

## 7. 테스트 시나리오

### 7.1 파서 레벨

- 2행 헤더 HTML 표 입력 시 merged header 생성 확인
- 동일 `rowspan` 헤더가 단일 헤더로 축약되는지 확인
- 중복 헤더명이 `_2`, `_3` suffix로 보존되는지 확인
- 단일 헤더 BOM 표가 무회귀인지 확인

### 7.2 데이터 정합성 레벨

- 실측 BOM 첫 행의 열 밀림이 해소되는지 확인
- `headers` 길이와 `rows[0].keys()`가 일치하는지 확인
- 앞쪽 셀 누락 케이스와 단순 끝쪽 padding 케이스를 구분 검증

### 7.3 Excel 레벨

- 시트명 `BOM_자재표` 인지성 유지 확인
- 첫 데이터 행이 실제 자재 데이터부터 시작하는지 확인
- JSON 열 수와 Excel 열 수가 일치하는지 확인

### 7.4 혼합 문서 / 경고 레벨

- 혼합 문서 경고 조건이 무오탐인지 확인
- 메타데이터가 있거나 표 수가 충분한 정상 BOM 문서는 경고하지 않는지 확인

### 7.5 자동 폴백 레벨

- 단위 테스트에서 `mixed_document` / `material_quote` 경고, `auto|always|never`, opt-out alias 동작을 검증
- CLI smoke에서 `_bom.*`와 `_bom_fallback_estimate.*` 생성 규칙을 검증
- 배치 집계는 fallback JSON이 있어도 manifest의 representative BOM만 집계하는지 검증

## 9. 한 줄 결론

지금 가장 먼저 고쳐야 할 것은 `메타 추출기`가 아니라 `복합 헤더 파서 + 데이터 행 정합성`이다.
그 다음이 회귀 테스트 고정과 문서/운영 기준 정리였고, 자동 폴백은 이제 `auto|always|never` 정책, manifest, representative/diagnostic 역할 분리까지 1차로 닫혔다. 이어서 estimate/pumsem/trade-statement/generic에도 최소 도메인 계약을 붙였으므로, 남은 큰 결정은 `DWG NO.` fallback, 거래명세서 preset, 듀얼 실행/머지, 구조 분해 순서다.
