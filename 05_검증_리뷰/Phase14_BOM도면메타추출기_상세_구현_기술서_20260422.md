## 0. 문서 목적

Phase 12.5가 Document Pipeline 쪽 Excel 렌더링 커버리지(Problem B)를 해결한 반면, **BOM Pipeline은 OCR 원문에 존재하는 도면 제목블록(Title Block) 메타데이터 — DWG NO, REV, TITLE, CLIENT, PROJECT, SCALE, DATE, 도면 작성/검도/승인자 등 — 를 파이프라인 어디에서도 보존하지 않고 버리는 구조**다.

`bom_state_machine.py`의 kill keyword 목록(`CLIENT:`, `PROJECT:`, `TITLE:`, `DRAWING NO`, `SCALE`, `DRW'D`, `CHK'D`, `APP'D` 등)이 **행 자체를 BOM 표 후보에서 제거**하므로 이 정보는 state machine 출력에는 없다. 다만 `BomExtractionResult.raw_text`에는 **OCR 전체 출력이 원형 그대로 보존**되어 있어 재OCR 없이 정규식 파싱만으로 복원 가능하다.

본 기술서는:
1. **실측 데이터 기반**으로 어떤 메타 필드가 실제로 어떤 포맷으로 OCR에 출력되는지 확인
2. 신규 모듈 `extractors/drawing_meta.py`의 API / 정규식 카탈로그 / 스키마 정의
3. `BomExtractionResult` 확장, `to_sections()` 주입점, 신규 Excel 시트 빌더까지 **전 파이프라인 연결선**을 확정
4. Step 단위(14-1 ~ 14-5) 커밋/태그/회귀 기준을 명시해 롤백 가능한 단위로 분할

한다.

## 1. 실측 — 현재 상태 (As-Is)

### 1.1 OCR 원문 실측 (`output/20260422_260421_견적(R0)_대산_..._bom.md`)

OCR 결과물에서 도면 메타가 실제로 어떻게 출력되는지 확인한 결과, **형태는 케이스별로 2가지**로 나뉨:

**케이스 A — HTML 테이블 cell (colspan 포함)**
```html
<tr><td colspan="3">CLIENT: 현대오일뱅크</td></tr>
<tr><td colspan="3">PROJECT: 대산 10TON CRANE 설치</td></tr>
<tr><td>TITLE: PIPE SUPPORT DETAIL</td><td>DWG NO. KO-D-010-14-16N</td><td>REV. 0</td></tr>
```

**케이스 B — 평문 라인 (Tesseract fallback 경로)**
```
DWG NO. KO-D-010-14-16N
REV. 0
TITLE: PIPE SUPPORT DETAIL
SCALE: N.T.S
DATE: 2026.04.21
```

공통 특성:
- **콜론(`:`) 또는 마침표(`.`) 구분자** — "CLIENT:" / "DWG NO." 혼재
- **공백 가변** — "DWG NO. KO-D-010" / "DWG  NO . KO-D-010" 둘 다 나타남
- **대소문자 혼재** — "Dwg No" / "DWG NO" / "dwg no" 모두 실측됨 (OCR 엔진별 차이)
- **한/영/일 혼재** — "CLIENT: 현대오일뱅크", "도면번호: KO-D-010" 패턴도 존재
- **누락 가능** — 도면 1장에 11개 필드 전부 있는 경우는 드묾. 평균 6~8개 존재

### 1.2 현재 코드 상태 실측

| 파일 | 현재 동작 | 유실 지점 |
|------|-----------|-----------|
| `extractors/bom_state_machine.py` (L26~219) | `presets/bom.py`의 kill keyword로 CLIENT/PROJECT/TITLE/DRAWING NO 포함 행을 필터 | **filter 후 버림** — 이 시점에서 메타 정보 소실 |
| `extractors/bom_types.py` (L29~51) | `BomExtractionResult`에 `raw_text` 보존 | **메타 전용 필드 없음** |

## 5. Step 분해

### Step 14-1 — `extractors/drawing_meta.py` 신규 작성

**목표**: raw_text 입력 → 11개 필드 dict 리스트 반환하는 순수 함수 모듈

**파일**: `ps-docparser/extractors/drawing_meta.py`

**공개 API**:
```python
def extract_drawing_meta(raw_text: str, default_page: int | None = None) -> list[dict]:
    """
    OCR 원문에서 도면 제목블록 메타데이터를 추출한다.

    Why: state machine이 kill keyword로 버린 정보를 복원.
         재OCR 없이 정규식 파싱만으로 처리.

    Args:
        raw_text: BomExtractionResult.raw_text 전체 문자열
        default_page: 페이지 구분이 없을 때 모든 dict에 부여할 기본값

    Returns:
        list[dict]: 페이지당 0~1개 dict. 필드값은 str | None.
                    모든 필드가 None이면 해당 dict 생략.
    """
```

**내부 구조**:
1. `_FIELD_PATTERNS: dict[str, list[re.Pattern]]` — 필드별 정규식 카탈로그 (IGNORECASE)
2. `_split_by_page(raw_text) -> list[tuple[int|None, str]]` — form feed (`\x0c`) 또는 `--- PAGE N ---` 마커 기준 분할
3. `_extract_one_page(text, page) -> dict | None` — 1페이지 분량 텍스트에서 11개 필드 시도
4. `_normalize(value) -> str` — strip, 내부 공백 정규화, 빈 문자열 → None 변환

**정규식 카탈로그 초안** (Step 14-1에서 확정):

| 필드 | 앵커 | 패턴 예시 |
|------|------|-----------|
| `dwg_no` | `DWG NO`, `DRAWING NO`, `도면번호` | `(?:DWG|DRAWING)\s*NO[\.:\s]+([A-Z0-9\-/]+)` |
| `rev` | `REV`, `REVISION` | `REV(?:ISION)?[\.:\s]+([A-Z0-9]+)` |
| `title` | `TITLE` | `TITLE[\.:\s]+(.+?)(?:\n|$)` |
| `date` | `DATE` | `DATE[\.:\s]+([\d\.\-/]+)` |
| `project` | `PROJECT` | `PROJECT[\.:\s]+(.+?)(?:\n|$)` |
| `client` | `CLIENT` | `CLIENT[\.:\s]+(.+?)(?:\n|$)` |
| `drawn_by` | `DRAWN`, `DRW'D`, `DRAWN BY` | `DRW'D|DRAWN(?:\s*BY)?[\.:\s]+([A-Za-z\.\s]+?)(?:\n|$)` |
| `checked_by` | `CHECKED`, `CHK'D` | 유사 패턴 |
| `approved_by` | `APPROVED`, `APP'D` | 유사 패턴 |
| `scale` | `SCALE` | `SCALE[\.:\s]+([A-Z0-9\.\:\s]+?)(?:\n|$)` |
| `sheet` | `SHEET` | `SHEET[\.:\s]+(\d+\s*(?:OF|/)\s*\d+)` |
