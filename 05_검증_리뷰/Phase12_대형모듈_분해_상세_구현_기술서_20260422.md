## 0. 문서 목적

Phase 10(테스트 커버리지 71.82% 달성)과 Phase 11(CI/CD 인프라 구축)이 완료되어 **회귀 안전망**이 충분해졌다. 이 기반 위에서 Phase 12는 **500줄 초과 모듈을 기능 단위로 분해**하여 가독성·변경 비용·테스트 경계성을 개선한다.

대형 모듈 분해는 본질적으로 **위험한 리팩터**다. 공개 API(import 경로, 함수 시그니처)가 외부 코드(`main.py`, `pipelines/`, 테스트 스위트)에 광범위하게 의존하기 때문이다. 본 기술서는 다음을 명시한다:

- 현재 대상 모듈의 **함수 단위 이동표** (target structure가 아닌 구체적 함수 배치)
- **Import Shim** 전략 — 기존 import 경로 100% 유지
- Step별 **Golden E2E + 552 baseline 회귀 체크포인트**
- 각 Step별 **롤백 단위**와 중단 지점 (Step 경계에서 Git 태깅)
- Mock/픽스처 재사용 원칙 — 기존 Phase 10 테스트 100% 재사용

## 11. 롤백·회귀 방지 전략

### 11.1 Step 경계 태깅

각 Step 착수 전 태그 부여:
```

```

실패 시 `git reset --hard phase12-stepN-start`로 즉시 복구.

### 11.2 Step별 승인 기준 (DoD)

한 Step이 "완료"로 간주되려면 **모든 CP 통과 + 다음 3가지 추가 검증**:

| # | 검증 | 실행 |
|---|---|---|
| G1 | `pytest --cov=. --cov-fail-under=70` | coverage gate 유지 |
| G2 | `ruff check .` + `ruff format --check .` | lint/format 클린 |
| G3 | Golden E2E (`tests/integration/test_golden_*.py`) | 전수 MATCH |

### 11.3 회귀 탐지 전략

- **pytest baseline**: 각 Step 전후 `pytest --tb=no -q | tail -1`로 passed 수 비교
- **Coverage delta**: 각 Step 전후 `coverage` 출력 커버리지 비교 — **-2%p 이상 하락 시 롤백 검토**
- **Golden hash**: `tests/integration/fixtures/golden/` JSON을 `git diff` — 0 byte 변화

### 11.4 중단 지점

임의 Step에서 막혔을 때 **즉시 중단 가능**. 각 Step은 독립적 가치를 가진다:
- Step 12-1만 완료해도 → table_parser 가독성 개선
- Step 12-1~12-2 완료해도 → 500줄 초과 1건 제거
- 12-4/12-5는 언제든 다음 Phase로 연기 가능
