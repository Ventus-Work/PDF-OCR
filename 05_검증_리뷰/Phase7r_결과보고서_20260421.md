## 9. 기술적 교훈

### 9.1 Lazy Import 엔진의 Mock 패턴

엔진 클래스가 `__init__` 안에서 `from X import Y` 를 수행할 때 모듈 레벨 patch 는 이미 완료된 import 를 되돌리지 못한다. 해결책:

```python
engine = GeminiEngine(api_key="fake", ...)
engine._genai = MagicMock()     # 생성 후 주입이 가장 견고
```

### 9.2 FREE_TIER_DELAY 제거

Gemini 엔진은 Free Tier 호출 간 `time.sleep` 로 4초 대기. 테스트에서는 단순 patch:

```python
mocker.patch("engines.gemini_engine.time.sleep", return_value=None)
```

6개 케이스 기준 **24초 단축**.

### 9.3 Windows 파일 권한 테스트

Windows 는 `os.chmod(dir, stat.S_IREAD)` 로 디렉터리 쓰기를 막지 못한다. 대신 `builtins.open` 자체를 patch 하여 `PermissionError` 를 직접 유발하는 방식으로 교정.

### 9.4 pytest import mode 주의

`tests/unit/<subdir>/__init__.py` 를 임의로 추가하면 기존 하위 디렉터리 관행과 충돌해 `ModuleNotFoundError` 가 발생. **하위 디렉터리는 `__init__.py` 를 두지 않는 pytest rootdir 패턴**을 일관되게 유지해야 한다.
