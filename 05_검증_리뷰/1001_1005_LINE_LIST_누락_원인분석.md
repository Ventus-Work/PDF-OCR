# 1001 / 1005 LINE LIST 누락 원인 분석

작성일: 2026-04-16

---

## 증상

`PIPE-PR2-PS-1001-S1-REV1` 및 `PIPE-PR2-PS-1005-S1-REV1` 도면에서
LINE LIST가 추출되지 않아 집계 결과에 누락됨.

---

## 원인: ZAI OCR의 희소(Sparse) 테이블 미감지

### ZAI 감지 결과 비교

| 구분 | OCR 출력 크기 | ZAI 감지 테이블 수 | LINE LIST 포함 |
|---|---|---|---|
| **1002 ~ 1009 (정상)** | 6,500 ~ 8,200 bytes | **3개** (BOM + LINE LIST + 개정이력) | ✅ |
| **1001 (문제)** | 1,566 bytes | **1개** (BOM만) | ❌ |
| **1005 (문제)** | 1,906 bytes | **1개** (BOM만) | ❌ |

---

### 핵심 차이: LINE LIST 데이터 행 수

ZAI의 layout 분석기는 테이블 내 **비어있지 않은 셀의 밀도**를 기준으로 테이블 여부를 판정한다.

```
1001 / 1005 LINE LIST (ZAI 미인식)        1002 ~ 1009 LINE LIST (ZAI 정상 인식)
┌────┬──────────────────┬──────┬──────┐   ┌────┬──────────────────┬──────┬──────┐
│ SN │    LINE NO.      │ SIZE │ ITEM │   │ SN │    LINE NO.      │ SIZE │ ITEM │
├────┼──────────────────┼──────┼──────┤   ├────┼──────────────────┼──────┼──────┤
│ A  │ 450-N2003-VG-... │ 450A │GUIDE │   │ A  │ 600-N2001-...   │ 600A │U-BOLT│
├────┼──────────────────┼──────┼──────┤   ├────┼──────────────────┼──────┼──────┤
│    │                  │      │      │   │ B  │ 450-N2003-...   │ 450A │GUIDE │
├────┼──────────────────┼──────┼──────┤   ├────┼──────────────────┼──────┼──────┤
│    │   (빈 행 8개)    │      │      │   │ C  │ 300-N2006-...   │ 300A │GUIDE │
│    │                  │      │      │   │    │  ... (3~9개 데이터 행) ...      │
└────┴──────────────────┴──────┴──────┘   └────┴──────────────────┴──────┴──────┘
  밀도 낮음 → ZAI가 테이블로 미인식           밀도 높음 → ZAI가 테이블로 정상 인식
```

- **1001/1005**: 해당 도면의 파이프 라인이 1개뿐 → LINE LIST 데이터 행 **1개** + 빈 행 8개
- **1002~1009**: 파이프 라인이 다수 → LINE LIST 데이터 행 **3~9개**

---

### 연쇄 실패 과정

```
[1차 OCR - 전체 페이지]
  ZAI layout 분석 → LINE LIST를 테이블로 미인식
  → OCR 출력에 BOM 테이블만 포함
  → BOM 빈 행들이 출력 버퍼 소진
  → LINE LIST 영역 도달 불가 (출력 truncation)

[3차 OCR - layout bbox 전체 크롭]
  타이틀 블록 전체를 크롭해도 동일하게 BOM에서 truncation 발생

[결과]
  LINE LIST = 0행
```

## 적용된 해결책

### 수정 파일 3개

| 파일 | 변경 내용 |
|---|---|
| `extractors/bom_extractor.py` | **4차 OCR 추가**: layout bbox 내 중간 구간(25%~72%)만 별도 크롭 → LINE LIST를 이미지 최상단에 배치하여 ZAI가 우선 인식하도록 유도 |
| `parsers/bom_table_parser.py` | **잘린 HTML 테이블 보완**: ZAI가 `</table>` 없이 출력을 끊으면 regex 매칭 실패 → `<table>` 시작 태그부터 끝까지를 불완전 블록으로 처리 |
| `exporters/bom_aggregator.py` | **Unicode 따옴표 정규화**: ZAI가 `MAT'L`, `Q'TY` 헤더에 U+2019(`'`) 사용 → ASCII 따옴표(`'`)로 정규화 후 alias 매핑 |

### 4차 OCR 동작 원리

```
layout_details의 table bbox (타이틀 블록 전체)
    x=6967~9737,  y=245~6836  (9925×7017 이미지 기준)

4차 크롭 = bbox 내 25%~72% 구간
    y = 245 + (6591 × 0.25) ~ 245 + (6591 × 0.72)
    y = 1892 ~ 4990

→ BOM 하단 빈 행 제외 + LINE LIST 헤더·데이터 포함 + 개정이력 제외
→ ZAI가 LINE LIST를 이미지 상단에서 최초 인식
```

이 비율은 **page 절대 좌표가 아닌 감지된 bbox 내 상대 비율**이므로
도면 레이아웃이 달라져도 적용 가능하다.
