# PDF-OCR / PJT_A

PJT_A는 PDF 견적서와 BOM 문서를 OCR, 구조화 JSON, Excel 산출물로 변환하는
`ps-docparser` 프로젝트와 그 운영 문서를 함께 보관하는 작업 폴더입니다.

## 폴더 지도

| 경로 | 용도 | 정리 기준 |
|---|---|---|
| `ps-docparser/` | 실제 사용 코드, 테스트, API/UI, 실행 스크립트 | 기능 변경과 정리 변경을 분리한다. |
| `00_견적서_원본/` | 원본 PDF/XLSX | Git 제외, 이동/삭제하지 않는다. |
| `01_OCR_결과물/` | OCR 결과 보관 | tracked 삭제 이슈는 기능 변경과 분리한다. |
| `02_기획_계획서/` | 기획, 통합, 검증 계획서 | 문서 보존. |
| `03_Phase별_기술서/` | Phase별 상세 구현 기술서 | 문서 보존. |
| `04_Phase별_결과보고서/` | Phase별 구현 결과 보고서 | 문서 보존. |
| `05_검증_리뷰/` | 검증, 리뷰, AgentLog, 변경묶음 기록 | 색인으로 찾기 쉽게 유지한다. |
| `06_이미지_스크린샷/` | 참조 이미지와 캡처 | 문서 증거로 보존. |
| `98_archive/` | 2026-04-30 이후 정리 아카이브 | Git 제외, 삭제 전 임시 보관소. |
| `99_legacy_OCR/` | 구버전 OCR 자료 | 기존 cold archive로 유지. |

## 현재 운영 원칙

- 루트는 안내 문서와 분류 폴더만 둔다.
- 코드는 `ps-docparser/` 안에서만 운영한다.
- `ps-docparser/output/`은 Git 제외 작업 폴더이며, 대표 증거 run만 남긴다.
- 반복 실행본, 서버 로그, 임시 로그는 먼저 `98_archive/`로 이동하고 바로 삭제하지 않는다.
- `.env`, `output/ui_usage/usage.db`, 변경묶음 manifest, 검증 보고서는 이동하지 않는다.
- 커밋, 푸시, 삭제는 별도 명시 요청 전에는 하지 않는다.
